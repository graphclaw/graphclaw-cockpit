# GraphClaw Cockpit Wireframe Review Checklist

## Iteration 1 scope
- Cockpit shell
- Task + explainability
- Agent monitor
- Canvas editor

## UX validation checklist

### 1) Navigation and IA
- [ ] Global navigation is clear within 5 seconds
- [ ] Users can identify where to do daily triage vs weekly planning
- [ ] Context rail concept is useful and not noisy

### 2) Task usability (Notion-like)
- [ ] Task blocks are readable without training
- [ ] Primary actions (Approve/Snooze/Delegate) are obvious
- [ ] Explainability table is understandable for non-technical users

### 3) Monitoring efficiency (Linear-like)
- [ ] Top KPI strip gives immediate state awareness
- [ ] Action queue table supports fast operational decisions
- [ ] Event feed and trigger schedule reduce context switching

### 4) Canvas ergonomics (Figma-like)
- [ ] Palette/canvas/inspector layout feels intuitive
- [ ] Users can infer drag-connect-configure flow
- [ ] Test panel output is understandable and confidence-building

### 5) Simplicity and adoption
- [ ] UI avoids overwhelming first-time users
- [ ] Advanced controls are discoverable but not dominant
- [ ] Language is plain and low-jargon in key decisions
