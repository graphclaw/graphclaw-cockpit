## Design System Rules

### Typography
- Primary font: Inter (loaded via Google Fonts CDN)
- Fallback: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
- Scale: 11px (micro), 12px (caption), 13px (body-sm), 14px (body), 16px (title), 20px (heading), 28px (display)
- Weights: 400 (regular), 500 (medium), 600 (semibold), 700 (bold)
- Line heights: 1.2 (headings), 1.5 (body), 1.6 (loose/reading)

### Color System
Semantic state palette (from PRD section 02):
- `--state-active: #3B82F6` (blue)
- `--state-progress: #10B981` (green)
- `--state-blocked: #EF4444` (red)
- `--state-delayed: #F59E0B` (amber)
- `--state-complete: #9CA3AF` (grey)
- `--state-snoozed: #E5E7EB` (light grey)
- `--state-review: #A855F7` (purple)

Neutral palette (light + dark mode):
- Background, surface, surface-alt, border, border-strong
- Text primary, secondary, tertiary, inverse

Brand accent: derive from the GraphClaw logo — inspect `docs/graphclaw.ai.png` for dominant color.

### Spacing
- Base unit: 4px
- Scale: 2px, 4px, 6px, 8px, 12px, 16px, 20px, 24px, 32px, 40px, 48px, 64px
- Use consistent spacing — no arbitrary values

### Elevation (Shadows)
- Level 0: none (flat)
- Level 1: `0 1px 2px rgba(0,0,0,.05)` (cards, subtle lift)
- Level 2: `0 2px 8px rgba(0,0,0,.08)` (dropdowns, popovers)
- Level 3: `0 8px 24px rgba(0,0,0,.12)` (modals, dialogs)
- Level 4: `0 16px 48px rgba(0,0,0,.16)` (floating panels)

### Border Radius
- Small: 4px (badges, pills)
- Medium: 6px (inputs, buttons)
- Large: 8px (cards)
- XL: 12px (modals, panels)
- Full: 9999px (circular avatars, round pills)

### Icons
- Use Lucide Icons via CDN: `https://unpkg.com/lucide@latest`
- Every icon must have `aria-label` for accessibility
- Size: 16px (inline), 20px (buttons), 24px (navigation)

### Dark/Light Mode
- Define ALL colors as CSS custom properties
- Toggle via `data-theme="dark"` on `<html>`
- Include a working JS toggle in every page

## Component Library (Build These)

### Buttons
- Primary (filled blue), Secondary (outlined), Ghost (text-only), Danger (red)
- States: default, hover, focus (ring), active, disabled, loading (spinner)
- Sizes: sm (28px), md (36px), lg (44px)

### Inputs
- Text input with label + optional helper text + error state
- Select dropdown, Multi-select with tags
- Checkbox, Radio, Toggle switch
- Slider (for scoring weights W1-W7)
- Date picker, Time picker
- Masked password input (for secrets)
- Textarea (for descriptions)
- Search input with icon

### Data Display
- Badge: state colors, sizes (sm/md), dot indicator variant
- Pill/Tag: removable, colored by category
- Card: with header, body, footer areas; elevation levels
- Table: sortable headers, row hover, row selection, inline actions, virtual scroll hint
- KPI card: value, label, trend indicator, sparkline placeholder
- Progress bar: percentage, color by status
- Avatar: initials fallback, size variants
- Tooltip: on hover, positioned auto
- Popover: click-triggered, with content area

### Navigation
- Sidebar: collapsible, with icon+label items, section dividers, active state
- Tabs: horizontal tab bar with underline indicator
- Breadcrumbs: with separator, clickable ancestors
- View switcher: 3-way toggle (Graph/Table/Dependencies)

### Feedback
- Toast: success/warning/error/info variants, auto-dismiss, action button
- Skeleton loader: card, table row, text line variants
- Empty state: icon + message + primary action button
- Loading spinner: inline and page-level
- Error state: message + retry button

### Overlays
- Modal: backdrop, close button, header/body/footer, trap focus
- Side panel: slide from right, 350px-480px width, header + scrollable body
- Dropdown menu: positioned auto, keyboard navigable
- Context menu: right-click triggered

## Anti-AI Rules (CRITICAL)

1. **No dashed placeholder borders** — use solid borders or shadows
2. **No annotation text explaining intent** — the design must speak for itself
3. **No generic placeholder data** — use realistic task names ("Finalize Q3 pricing model", "Review Kubernetes migration plan"), real-looking dates, real org names ("Product Ops", "Infrastructure", "Growth")
4. **No uniform spacing** — vary visual rhythm; use tighter spacing for related items, more space between sections
5. **No everything-looks-the-same** — buttons look different from badges, badges look different from tags, tags look different from status indicators
6. **No single-pattern layouts** — mix card grids with tables, mix full-width with constrained-width, use asymmetric splits where appropriate
7. **Use visual hierarchy aggressively** — the most important thing must be visually dominant (size, weight, position, color)
8. **Add micro-interactions** — CSS transitions on hover (transform, shadow, color), smooth panel open/close, subtle opacity changes
9. **Empty states must have personality** — illustration placeholder + helpful message + clear action ("No tasks yet. Create your first task to get started.")
10. **Dark mode must be excellent** — not just inverted colors; adjust shadows, reduce contrast slightly, use slightly desaturated colors

## Aesthetic Direction (Module-Based)

| Module | Style Reference | Key Traits |
|--------|----------------|------------|
| Task views & blocks | Notion | Clean card blocks, subtle dividers, readable hierarchy, spacious |
| Settings & operations | Linear | Dense but elegant, bold labels, tight spacing, keyboard-first feel |
| Canvas editor | Figma | Precise controls, floating toolbars, infinite canvas feel, grid snapping |
| Onboarding & empty states | Canva | Guided flows, friendly illustrations, warm gradients, step-by-step |
| Shell & navigation | Linear + Notion | Icon sidebar, clean top bar, command palette affordance |

## Responsive Rules
- Mobile (< 768px): single column, bottom tab nav, simplified views, 44px min touch targets
- Tablet (768-1024px): split views, collapsible panels, medium density
- Desktop (> 1024px): full persistent panels, maximum density, keyboard shortcuts visible
