# Governance

## Roles

- Triager: labels issues, asks clarifying questions, closes duplicates.
- Reviewer: reviews pull requests and proposes changes.
- Maintainer: merges pull requests and manages releases.
- Admin: repository settings, permissions, and incident handling.

## Promotion Path

- Contributor -> Triager: consistent high-quality issue triage.
- Triager -> Reviewer: reliable technical feedback over multiple PRs.
- Reviewer -> Maintainer: sustained contributions and release readiness.

## Decision Process

GraphClaw uses lazy consensus for most decisions:

- Propose change in issue/PR/discussion.
- Allow time for maintainer review.
- If no substantial objection, proceed.

For security-sensitive or breaking changes, maintainer approval is required.

## Release Cadence

- Patch releases: as needed.
- Minor releases: approximately every 4 weeks.
